# CTF «Шифрование» — План проекта

## 1. Общая идея проекта

Проект представляет собой CTF-задачу в формате **последовательной цепочки загадок**, посвящённых теме **шифрования и кодирования информации**.

Продуктом проекта является **статический веб-сайт**, размещаемый на GitHub Pages.  
Серверная часть отсутствует. Все задания решаются анализом контента страниц сайта.

Игроку доступен **только веб-интерфейс**.  
Исходный код сайта может быть доступен, но не должен содержать явных ответов.

---

## 2. Механика прохождения

Прохождение осуществляется поэтапно.

- Каждый этап представляет собой отдельную страницу сайта.
- Каждый этап содержит:
  - описание задания;
  - зашифрованные данные (текст, строки, таблицы и т.д.);
  - требования к формату ответа;
  - поле ввода ответа (или производного от ответа значения).

### Формат ответа

- Ответом всегда является **строка**.
- В ответах не используются пробелы.
  - Если требуется несколько слов — используется символ `_`.
- По умолчанию используется латиница.
- Регистр не важен, если не указано иное.

### Переход между этапами

- Прямой нумерации этапов для игрока нет.
- Для перехода на следующий этап используется **токен**, вычисляемый из ответа.
- Токен формируется по единому правилу (например, SHA-256 от ответа с усечением).
- Следующая страница доступна только при вводе корректного токена.

---

## 3. Требования к безопасности (в рамках статического сайта)

- Ответы не должны храниться в явном виде в HTML или JavaScript.
- Проверка должна осуществляться через производные значения (хэши, вычисления).
- Но, абсолютная защита не требуется.

---

## 4. Структура заданий (10 этапов)

Сложность этапов возрастает последовательно: от простых кодировок к комбинированным криптографическим задачам.

---

### Этап 1 — Кодирование (Base64)

**Цель:**  
Познакомить игрока с форматом заданий и механикой токенов.

**Задание:**  
Дана строка в кодировке Base64. Требуется декодировать её и получить осмысленное слово или фразу.

**Ожидаемый ответ:**  
- одно слово;
- латиница;
- без пробелов.

---

### Этап 2 — Цезарь / ROT13

**Цель:**  
Проверка знания простейших симметричных подстановок.

**Задание:**  
Дан текст, зашифрованный сдвигом Цезаря (ROT13).  
Требуется восстановить исходный текст и извлечь ключевое слово.

**Ожидаемый ответ:**  
- одно слово;
- латиница.

---

### Этап 3 — Атбаш

**Цель:**  
Работа с классическим зеркальным шифром подстановки.

**Задание:**  
Дан текст, зашифрованный методом Атбаш.  
Требуется восстановить исходное слово.

**Ожидаемый ответ:**  
- одно слово;
- латиница.

---

### Этап 4 — Частотный анализ (упрощённый)

**Цель:**  
Научить игрока распознавать шифр по структуре текста.

**Задание:**  
Дан короткий зашифрованный текст.  
Известно, что это простой подстановочный шифр (вариация Цезаря).

Игрок должен определить сдвиг и восстановить читаемую фразу, из которой извлекается ответ.

**Ожидаемый ответ:**  
- одно слово;
- латиница.

---

### Этап 5 — Шифр Виженера

**Цель:**  
Показать работу полиалфавитного шифра.

**Задание:**  
Дан шифротекст Виженера.  
Ключ шифрования скрыт непосредственно на странице (например, акростихом, визуальным паттерном или структурой текста).

**Ожидаемый ответ:**  
- одно слово;
- латиница.

---

### Этап 6 — XOR-шифрование

**Цель:**  
Познакомить с побитовым шифрованием.

**Задание:**  
Даны:
- ключ (строка);
- данные в шестнадцатеричном представлении.

Требуется выполнить XOR по байтам с повторяющимся ключом и получить читаемое слово.

**Ожидаемый ответ:**  
- одно слово;
- латиница.

---

### Этап 7 — Комбинация значений + хэш

**Цель:**  
Научить игрока работать с криптографическими хэш-функциями.

**Задание:**  
Игроку даны несколько значений, полученных на предыдущих этапах.  
Необходимо собрать строку в указанном формате и вычислить от неё хэш.

**Ожидаемый ответ:**  
- строка фиксированной длины;
- шестнадцатеричное представление.

---

### Этап 8 — Многоуровневое кодирование

**Цель:**  
Развить навык распознавания вложенных преобразований.

**Задание:**  
Дана строка, закодированная **несколькими слоями** (например, Base64 → Base64).  
Игрок должен определить количество слоёв и получить осмысленный результат.

**Ожидаемый ответ:**  
- одно слово;
- латиница.

---

### Этап 9 — Логическое извлечение данных

**Цель:**  
Показать, что не каждое задание является криптографическим алгоритмом.

**Задание:**  
На странице представлена таблица или список значений.  
Требуется извлечь информацию по логическому правилу (порядок, первые буквы, индексы и т.п.).

**Ожидаемый ответ:**  
- короткая строка;
- латиница.

---

### Этап 10 — Финальный комбинированный замок

**Цель:**  
Проверить способность игрока комбинировать несколько техник.

**Задание:**  
Игрок должен:
1. Использовать ответы нескольких предыдущих этапов.
2. Собрать строку по заданному шаблону.
3. Применить криптографическое преобразование (хэширование, кодирование).
4. Использовать полученный результат как ключ для расшифровки финального сообщения.

Результатом является 'флаг'.

**Ожидаемый ответ:**  
- Строка с поздравлениями о завершении испытания.
